Detailed Description

The Process Event Queue (peq) module deals with the events that command the simulation.

Different type of events can coexist in the queue. The Process Event Queue (peq) module is responsible for storing these events and allowing them to be accessed in chronological order.

It is up to the implementer to decide which data structure to use, as long as, at least, the following information is present:

  • the queue of events, that must be kept sorted in ascending order of event time and, for events with the same time stamp, in order of insertion. The use of a sort algorithm is not allowed.
  • the current time, that corresponds to the time of the last event fetched.

The interface of this module is predefined, being composed of the following functions:

function function ID role
peqInit() 301 Initialize the support internal data structure;
peqInsert() 304 Insert a new event in the queue;
peqLog() 302 Log the internal state of the process event queue.
peqIsEmpty() 303 Indicate if the queue is empty;
peqPeekNext() 305 Peek the front event;
peqFetchNext() 306 Fetch the front event;
peqEventTypeAsString() 307 Return an event type as a string

Enumerator
ARRIVAL 

A process arrives to the system.

WAKEUP 

A blocked process wakes up from its I/O-burst.

BLOCK 

A running process blocks after one of its (not last) CPU-burst.

TERMINATE 

A running process terminates after its last CPU-burst.

uint32_t  time,
uint32_t  pid 
)

Inserts an entry in queue.

A new entry should be created and added to the process event queue.
The following must be considered:

  • If pid is 0, the EINVAL exception must be thrown.
  • If time is not in the future, the EINVAL exception must be thrown.
  • All exceptions must be of the type defined in this project (Exception).
Parameters
[in]typeThe type of event to be inserted (one of ARRIVAL, POSTPONED, or TERMINATE)
[in]timeTime at which event will occur
[in]pidId of the process associated to the event

◆ peqPeekNext()

Event peqPeekNext ( )

Peek the event in the front of the queue.

The event to be selected is the first in the queue, which, as said before, is kept properly sorted. The following must be considered:

  • The event must not be removed from the queue.
  • The EINVAL exception should be thrown, if no event exists.
  • All exceptions must be of the type defined in this project (Exception).
Returns
The front event in the queue

◆ peqFetchNext()

Event peqFetchNext ( )

Fetch the event in the front of the queue.

The event to be selected is the first in the queue, which, as said before, is kept properly sorted. The following must be considered:

  • The current time must be set to the time of the event fetched.
  • The event must be removed from the queue.
  • The EINVAL exception should be thrown, if no event exists.
  • All exceptions must be of the type defined in this project (Exception).
Returns
The front event in the queue
